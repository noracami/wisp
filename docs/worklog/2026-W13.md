## 2026-03-24

### 完成：部署上線 + CI/CD + Discord 端對端測試通過

#### 部署架構

```
git push → GitHub Actions (build image)
  → GCP Artifact Registry (push image)
  → Deploy Webhook on VM (pull & restart)
  → Discord 通知

Internet → Cloudflare Tunnel → wisp:8080 (GCP VM)
```

#### 基礎建設

- **GCP VM**（e2-small, Ubuntu 24.04, asia-east1-b）
  - Docker 29.3.0 + Docker Compose 5.1.1
  - 4G swap（Rust build 用，雖然最後改用 GHA build）
  - Deploy key 設定完成（唯讀，僅限 wisp repo）
  - 程式碼放在 `/opt/wisp`
- **Cloudflare Tunnel** — `wisp.miao-bao.cc`
  - `/hooks/*` → deploy webhook (port 9000)
  - 其餘 → wisp service (port 8080)
- **GCP Artifact Registry** — `asia-east1-docker.pkg.dev/careful-broker-485510-r0/wisp/wisp`
  - Workload Identity Federation（無需 JSON key）

#### CI/CD Pipeline

- **GitHub Actions**：build image → push Artifact Registry → call deploy webhook → Discord 通知
- **Deploy Webhook**：VM 上跑 `webhook` (adnanh/webhook) 作為 systemd 服務，收到 POST + token 驗證後執行 `docker compose pull && up -d`
- **通知**：部署完成後發 Discord embed（成功/失敗）

#### Discord Bot

- Interactions Endpoint 設定完成：`https://wisp.miao-bao.cc/discord/interactions`
- Bot 已加入 Server，Slash Command `/chat` 已註冊（Guild scope）
- **端對端測試通過** — `/chat message:hello` → Wisp 回覆 "Hello! 👋 How can I help you today?"
- Server 內 ephemeral 回覆（只有自己看到），DM 公開回覆

#### 建立的 Skill

- **discord-commands** — 管理 Discord Slash Commands 的 CRUD skill

#### 踩過的坑

1. **Dockerfile Rust 版本** — `rust:1.85` 太舊，dependency 需要 1.88+
2. **Dockerfile 缺 OpenSSL** — 需要 `pkg-config` + `libssl-dev`
3. **VM 上 build 太慢** — e2-small 只有 1 vCPU，改用 GitHub Actions build
4. **Migration 不冪等** — `CREATE TABLE` 沒有 `IF NOT EXISTS`，每次重啟都衝突
5. **Migration 001 vs 002 衝突** — 001 的舊 schema index 引用不存在的欄位，改成 001 只保留 extensions
6. **Watchtower 無法拉 Artifact Registry** — Docker credential helper 用 `gcloud`，container 內沒有。改用 deploy webhook 方案
7. **Discord 通知 JSON 壞掉** — commit message 含換行，用 `jq` 正確 escape
